# End-to-End Test Harness Plan (the "test everything" capstone)

> **For Claude:** Build AFTER C-1 (debug track) lands + is reviewed. The existing 940+ vitest tests are unit/component-level with mocked ports. This adds a true END-TO-END layer: the real `Controller` + real phase/track classes + real engine (FSM, sizing, lifecycle, action-monitor, partitioner) driven by a **scripted stub host** — proving the whole pipeline wires together without model billing. (Note for the operator: pytest/cypress don't apply — TS/vitest stack, CLI extension, no web UI, no Python.)

**Why:** Every increment was reviewed-to-zero at the unit level, but the integration tests mock the host LLM per-phase. A bug in how phase outputs CHAIN (P1→P2→…→commit, or seed→P4→P5, or D1→…→D5) across the real controller loop could pass every unit test and still break a live run. The e2e harness is the deterministic proof of the full chain.

## Architecture: the scripted stub host
A `StubHost` replaces the real Claude steer. On each `hostAgent.steer({expectFile})`, instead of a model call, it:
1. Looks up a canned, schema-VALID response for the current phase/step (keyed by `expectFile` or phase name).
2. Writes that JSON to the `expectFile` path (the real phase-output validators then accept it).
3. Fires `agent_end` so the controller's steer resolves.
This makes a full run deterministic + fast + billing-free, while exercising the REAL: `_onInput` → resolver → sizing → `_runPhases`/`_runPhasesQuick`/`_runDebugTrack` → FSM advance → compaction (no-op on small ctx) → action-monitor write-confinement → gitOps → lifecycle release. Only the LLM is stubbed; everything else is real.

The stub uses a TEMP git repo as the project dir (real gitOps commit/push to a local bare remote, or a mock gitOps that records calls). Real `BoundedExec` for the debug track's repro gate (with canned tiny vitest repro files that genuinely go red→green), or a scripted BoundedExec stub returning red×3 then green×3.

## Scenarios to cover (each: drive an idea, assert the terminal state)
1. **Full build pipeline** (no prefix, sized to M/L): idea → P1..P6 → a real commit SHA on the temp repo; assert all 6 phase-output files exist + valid, FSM reached P6, lock released, cwd restored.
2. **Quick gear** (`quick:` → XS): idea → seed→P4→P5→P6 → commit; assert P1/P2/P3 ceremony SKIPPED (no discovery/panel steers), tests-gate still enforced, commit made.
3. **Middle gear** (`mid:`): P1→P3→P4→P5→P6, P2 skipped.
4. **Phase-by-phase** (`step:`): with a stub `ctx.ui.select` returning 'continue' each gate → full run completes; returning 'stop' at P3 → halts + lock released.
5. **Intent gate** (new project, `hasUI:true`, stub `ctx.ui.input`): asks 3 Qs → answers threaded into P1 → run proceeds.
6. **Debug track** (`debug:`): bug idea → D1 (BoundedExec red×3) → D2 → D3 (fix, changedFiles excludes repro) → D4 (green×3 + suite green) → D5 commit; assert the build pipeline P1-P6 NOT entered, lock released. Plus the failure paths: no-repro → escalate; D4 non-convergence → operator brief.
7. **Self-steer isolation**: fire an `input` with `source:'extension'` mid-run → ignored (no bogus run).
8. **Write confinement**: assert a phase that tries to write outside repoRoot is denied by the action-monitor (drive a stub host response with an out-of-root path → blocked).

## Acceptance
- A `tests/e2e/full-pipeline.e2e.test.ts` (+ maybe per-track files) drives each scenario through the REAL controller with the StubHost, asserting the terminal artifacts. No mocked phase classes — only the host LLM + (optionally) gitOps/boundedExec stubbed.
- Deterministic (reject-loud polls, B1 teardown-settle, temp-repo isolation, 10x green).
- tsc clean; full suite green; the e2e file runs in reasonable time (stub host = instant).

## Notes
- This is the closest thing to the live-run validation that's been gated on model billing — it proves the wiring the unit mocks can't.
- If a scenario reveals a real chaining bug, that's a finding → fix through the normal flow before declaring done.
- Keep the StubHost in a shared `tests/e2e/stub-host.ts` helper.
